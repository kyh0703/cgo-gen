# Documentation Map

This repository keeps durable project documentation under `docs/`.

## Core docs

- `ARCHITECTURE.md`: system shape, generation layers, and facade/model routing direction
- `PRODUCT.md`: product framing, goals, and non-goals

## Design docs

- `design-docs/`: longer-lived design and migration docs such as wrapping package planning
- `superpowers/specs/`: focused feature design specs used by active implementation work

## Planning and status

- `roadmaps/current-roadmap.md`: current implementation roadmap and near-term milestones
- `status/`: point-in-time status reports and adoption checkpoints
- `superpowers/plans/`: implementation plans that reference accepted specs

## References

- `references/`: CLI, config, IR, limitations, and hardening notes
